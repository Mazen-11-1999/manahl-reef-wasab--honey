# النشر (الإطلاق الفعلي) — مناحل ريف وصاب

دليل مختصر لنشر المتجر على سيرفر حقيقي وجعله جاهزاً للاستخدام.

---

## 1. متغيرات البيئة للإنتاج

انسخ `.env.example` إلى `.env` وعدّل القيم التالية عند النشر:

| المتغير | التطوير | الإنتاج (مثال) |
|--------|---------|-----------------|
| `NODE_ENV` | `development` | `production` |
| `PORT` | `3000` أو `3001` | حسب الاستضافة (مثلاً `3000`) |
| `FRONTEND_URL` | `http://localhost:3000` | `https://yourdomain.com` |
| `MONGODB_URI` | `mongodb://localhost:27017/...` | رابط Atlas أو استضافة MongoDB (من لوحة التحكم) |
| `JWT_SECRET` | أي قيمة للتجربة | سلسلة عشوائية طويلة (مثلاً 64 حرفاً) |
| `JWT_REFRESH_SECRET` | أي قيمة للتجربة | سلسلة عشوائية طويلة مختلفة |
| `ADMIN_PASSWORD` | كلمة تجريبية | **كلمة مرور قوية للمشرف** — غيّرها فور النشر |

- لا ترفع ملف `.env` إلى Git (يجب أن يبقى على السيرفر فقط).
- في الإنتاج استخدم **HTTPS** (شهادة SSL) إن أمكن.

---

## 2. تشغيل التطبيق

```bash
# تثبيت الاعتماديات (مرة واحدة)
npm install

# تشغيل السيرفر
# تطوير (مع إعادة تشغيل تلقائي):
npm run dev

# إنتاج (تشغيل مباشر):
NODE_ENV=production node server.js
# أو باستخدام pm2:
# pm2 start server.js --name manahl-badr
```

---

## 3. استضافة مقترحة

- **VPS** (مثل: DigitalOcean، Linode، استضافة محلية): ثبّت Node.js و MongoDB (أو استخدم MongoDB Atlas)، ثم شغّل التطبيق كما أعلاه.
- **MongoDB Atlas**: أنشئ Cluster واتصل به عبر `MONGODB_URI` في `.env`؛ لا تحتاج تثبيت MongoDB على نفس السيرفر.
- **الوصول من الإنترنت**: افتح المنفذ `PORT` في الجدار الناري؛ إن كنت تستخدم **Nginx** أو **Caddy** كـ reverse proxy، وجّه الدومين إلى `http://localhost:PORT`.

---

## 4. بعد النشر

1. غيّر **كلمة مرور المشرف** من لوحة التحكم (أو عبر `.env` ثم إعادة التشغيل إن كان التطبيق يقرأها عند البدء).
2. تأكد أن **لوحة التحكم** و**صفحات المتجر** تفتح عبر نفس الدومين (أو الفرعي) الذي وضعته في `FRONTEND_URL`.
3. راجع **طرق الدفع** و**معرض قصتنا** و**العملاء والشارات** من لوحة التحكم وتأكد أن البيانات تظهر بشكل صحيح.

---

## 5. ملخص سريع

- ضبط `.env`: `NODE_ENV=production`, `FRONTEND_URL`, `MONGODB_URI`, `JWT_*`, `ADMIN_PASSWORD`.
- تشغيل: `npm install` ثم `node server.js` (أو `pm2 start server.js`).
- استخدام HTTPS والدومين الصحيح في `FRONTEND_URL`.
- تغيير كلمة مرور المشرف بعد أول نشر.
